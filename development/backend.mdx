## Layered architecture

### Controller layer

Controllers handle HTTP requests and define REST endpoints:

```java
@RestController
@RequestMapping("/api/account-books")
@CrossOrigin
public class AccountBookController {
    private final AccountBookService accountBookService;

    public AccountBookController(AccountBookService accountBookService) {
        this.accountBookService = accountBookService;
    }

    @Operation(summary = "Get all account books")
    @GetMapping
    public List<AccountBookResponse> getAllAccountBooks() {
        return accountBookService.getAllAccountBooks();
    }

    @Operation(summary = "Create account book")
    @PostMapping
    public void createAccountBook(@RequestBody AccountBookRequest request) {
        accountBookService.createAccountBook(request);
    }

    @Operation(summary = "Update account book")
    @PutMapping("/{id}")
    public void updateAccountBook(
        @PathVariable Long id,
        @RequestBody AccountBookRequest request
    ) {
        accountBookService.updateAccountBook(id, request);
    }

    @Operation(summary = "Delete account book")
    @DeleteMapping("/{id}")
    public void deleteAccountBook(@PathVariable Long id) {
        accountBookService.deleteAccountBook(id);
    }
}
```

<Note>
All controllers use `@CrossOrigin` to enable CORS for frontend communication.
</Note>

### Service layer

Services contain business logic and coordinate between controllers and repositories:

```java
@Service
public class AccountBookService {
    private final AccountBookRepository accountBookRepository;
    private final ActionRepository actionRepository;
    private final CategoryRepository categoryRepository;

    public AccountBookService(
        AccountBookRepository accountBookRepository,
        ActionRepository actionRepository,
        CategoryRepository categoryRepository
    ) {
        this.accountBookRepository = accountBookRepository;
        this.actionRepository = actionRepository;
        this.categoryRepository = categoryRepository;
    }

    public List<AccountBookResponse> getAllAccountBooks() {
        return accountBookRepository.findAll().stream()
            .map(this::convertToResponse)
            .collect(Collectors.toList());
    }

    public AccountBookResponse createAccountBook(AccountBookRequest request) {
        AccountBook accountBook = new AccountBook();
        accountBook.setDate(request.getDate());
        accountBook.setAmount(request.getAmount());
        accountBook.setDescription(request.getDescription());
        
        accountBook.setAction(actionRepository.findById(request.getActionId())
            .orElseThrow(() -> new ResponseStatusException(
                HttpStatus.NOT_FOUND, "Action ID not found"
            )));
        
        accountBook.setCategory(categoryRepository.findById(request.getCategoryId())
            .orElseThrow(() -> new ResponseStatusException(
                HttpStatus.NOT_FOUND, "Category ID not found"
            )));
        
        AccountBook saved = accountBookRepository.save(accountBook);
        return convertToResponse(saved);
    }

    private AccountBookResponse convertToResponse(AccountBook accountBook) {
        return new AccountBookResponse(
            accountBook.getId().toString(),
            accountBook.getDate().toString(),
            accountBook.getAmount(),
            accountBook.getDescription(),
            accountBook.getAction().getActionDescription(),
            accountBook.getCategory().getCategoryDescription()
        );
    }
}
```

## Configuration

### Application properties

Configure the application in `src/main/resources/application.properties`:

```properties
# Database configuration
spring.datasource.url=jdbc:postgresql://localhost:5432/easyact
spring.datasource.username=postgres
spring.datasource.password=postgres

# JPA/Hibernate configuration
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect

# Server configuration
server.port=8080

# OpenAPI documentation
springdoc.api-docs.path=/api-docs
springdoc.swagger-ui.path=/swagger-ui.html
```

### OpenAPI configuration

Swagger documentation is configured in `OpenAPIConfig.java`:

```java
@Configuration
public class OpenAPIConfig {
    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("EasyACT API")
                .version("1.0.0")
                .description("Personal finance management API")
            );
    }
}
```

Access Swagger UI at: `http://localhost:8080/swagger-ui.html`

## Error handling

The application uses Spring's `ResponseStatusException` for error handling:

```java
public void deleteAccountBook(Long id) {
    if (!accountBookRepository.existsById(id)) {
        throw new ResponseStatusException(
            HttpStatus.NOT_FOUND,
            "AccountBook ID " + id + " not found"
        );
    }
    accountBookRepository.deleteById(id);
}
```

## Best practices

### Dependency injection

Use constructor-based dependency injection (recommended by Spring):

```java
@Service
public class AccountBookService {
    private final AccountBookRepository repository;
    
    // Constructor injection
    public AccountBookService(AccountBookRepository repository) {
        this.repository = repository;
    }
}
```

### DTO pattern

Always use DTOs to separate internal models from API contracts:

```java
public record AccountBookRequest(
    LocalDateTime date,
    Double amount,
    String description,
    String actionId,
    String categoryId
) {}

public record AccountBookResponse(
    String id,
    String date,
    Double amount,
    String description,
    String action,
    String category
) {}
```

### Transaction management

Use `@Transactional` for operations that require atomicity:

```java
@Transactional
public void deleteAccountBooksBatch(List<Long> ids) {
    List<AccountBook> accountBooks = accountBookRepository.findAllById(ids);
    if (accountBooks.size() != ids.size()) {
        throw new ResponseStatusException(
            HttpStatus.NOT_FOUND,
            "Some AccountBook IDs not found"
        );
    }
    accountBookRepository.deleteAllInBatch(accountBooks);
}
```

<Info>
Transactional methods automatically rollback on exceptions, ensuring data consistency.
</Info>
