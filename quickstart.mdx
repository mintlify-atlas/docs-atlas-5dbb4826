## Prerequisites

Before you begin, make sure you have the following installed on your system:

<CardGroup cols={2}>
  <Card title="Docker" icon="docker" href="https://www.docker.com/">
    Container runtime (version 20.10 or higher recommended)
  </Card>
  <Card title="Docker Compose" icon="layer-group" href="https://docs.docker.com/compose/">
    Multi-container orchestration tool (included with Docker Desktop)
  </Card>
</CardGroup>

<Note>
  If you're using Docker Desktop, Docker Compose is already included. Otherwise, install it separately following the official documentation.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the EasyACT repository to your local machine:

    ```bash
    git clone https://github.com/RL-Small-Project/EasyACT.git
    cd EasyACT
    ```
  </Step>

  <Step title="Start the services">
    Launch all services with a single command:

    ```bash
    docker-compose up -d
    ```

    This command will:
    - Pull the PostgreSQL 15 image
    - Build the Spring Boot API container
    - Build the Vue.js web application container
    - Create a private network for inter-service communication
    - Start all three services in detached mode

    <Note>
      The first startup may take several minutes as Docker builds the images and downloads dependencies.
    </Note>
  </Step>

  <Step title="Verify the services">
    Check that all containers are running:

    ```bash
    docker-compose ps
    ```

    You should see three containers running:
    - `easy_act-web` (port 8000)
    - `easy_act-api` (port 8080)
    - `easy_act-db` (port 5432)
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    ```
    http://localhost:8000
    ```

    You should see the EasyACT login page. You're ready to start tracking your finances!
  </Step>
</Steps>

## Configuration

The default configuration uses the following settings:

<CodeGroup>

```yaml docker-compose.yml
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres_password
      POSTGRES_DB: easy_act
    volumes:
      - ./data:/var/lib/postgresql/data

  api:
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://db:5432/easy_act
      SPRING_DATASOURCE_USERNAME: postgres
      SPRING_DATASOURCE_PASSWORD: postgres_password

  web:
    ports:
      - "8000:80"
```

</CodeGroup>

<Warning>
  The default database credentials are for development only. Change them before deploying to production.
</Warning>

## Managing the application

### View logs

To view logs from all services:

```bash
docker-compose logs -f
```

To view logs from a specific service:

```bash
# Web application logs
docker-compose logs -f web

# API server logs
docker-compose logs -f api

# Database logs
docker-compose logs -f db
```

### Stop the application

To stop all services while preserving data:

```bash
docker-compose stop
```

### Restart the application

To restart all services:

```bash
docker-compose restart
```

### Shut down and remove containers

To stop and remove all containers (data in `./data` directory is preserved):

```bash
docker-compose down
```

<Warning>
  To remove all data including the database, run `docker-compose down -v` or manually delete the `./data` directory. This action cannot be undone.
</Warning>

## Data persistence

Your financial data is stored in the `./data` directory on your host machine. This directory is mounted into the PostgreSQL container, ensuring your data persists even when containers are stopped or removed.

```
EasyACT/
├── api/
├── web/
├── docker-compose.yml
└── data/                    # Database files stored here
    └── postgresql/
```

<Note>
  Make sure to back up the `./data` directory regularly to prevent data loss.
</Note>

## Troubleshooting

### Port already in use

If you see an error about ports 8000 or 8080 being already in use, you can either:

1. Stop the service using that port
2. Modify the port mapping in `docker-compose.yml`:

```yaml
services:
  web:
    ports:
      - "3000:80"  # Changed from 8000 to 3000
```

### Containers fail to start

Check the logs for specific error messages:

```bash
docker-compose logs
```

Common issues:
- Insufficient disk space
- Docker daemon not running
- Previous containers not fully removed

### Reset the application

To completely reset EasyACT and start fresh:

```bash
# Stop and remove containers
docker-compose down

# Remove database files
rm -rf ./data

# Rebuild and start
docker-compose up -d --build
```
